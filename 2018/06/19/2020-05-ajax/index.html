ajax请求

一个ajax请求的步骤，异步处理，状态码，兼容处理，发送Get/Post，ajax+php登录

ajax是什么

　ajax是js和php等后端交互的手段；
　js请求一个后端的地址，后端返回的内容会到js，如果想让内容显示再页面上，那么就需要开 js 去渲染，比如：document.write，innerHTML，innerText，appendChild
　ajax 四个字母的含义：async + javascript + and + xml

发送一个ajax请求

分为4个步骤

  1. 创建一个 ajax 对象
    • var xhr = new XMLHttpRequest。实例化一个构造函数, 得到一个对象，得到的这个对象就能帮我们发送 ajax 请求；
  2. 配置本次请求的信息
    • xhr.open(请求方式, 请求地址, 是否异步)。请求方式: GET/POST/PUT；请求地址: 你要请求哪一个 php 文件；是否异步: 默认是异步的, 可以改成同步；
  3. 把这个请求发送出去
    • xhr.send()。这个方法一执行, 本次 ajax 请求就发送出去，后端接口收到；
  4. 接受响应
    • 后端返回的内容不会直接显示在浏览器上，而是给了 js 或者说给了 xhr 对象，需要拿到这个响应内容(后端给前端的数据)
    • 怎么接受响应？xhr.onload = function () {}。这个事件是 ajax 完成的时候触发，当本次请求完成以后, 就会触发后面的函数，xhr这个对象里面的responseText就是响应体

ajax的异步

　在一个ajax请求中，从第三步开始后变为异步；
　但第二步open的参数为false时，第三步为同步，则必须把onload事件提前，但一般不允许出现这种情况；

ajax的状态码

　http响应状态码：表示本次请求时成功还是失败；
　ajax状态码：描述当前处于ajax的哪一个步骤；通过ajax对象.readyState来获取状态码；
　当ajax的状态码就会改变时，会触发onreadystatechange事件；；
常见状态码：

  • 0: 表示你创建一个 ajax 对象成功了，var xhr = new XMLHttpRequest()
  • 1: 表示配置请求信息成功了(也就是执行完 open)，xhr.open('GET', './ajax.php')
  • 2: 表示响应体已经回到了浏览器, 但是还不能使用(因为此时还是一个响应报文)，xhr.send()
  • 3: 浏览器正在解析响应报文(把响应报文里面的响应体分离出来)，xhr.send()
  • 4: 浏览器分离响应体成功，已经可以正常使用了(本次请求结束了)，xhr.send()

ajax的兼容处理

　兼容IE678浏览器；ajax 有两个地方不兼容

  1. 创建 ajax 对象的时候不兼容
    • new XMLHttpRequest => 标准浏览器使用
    • new ActiveXObject => IE 低版本使用
  2. 接受响应的不兼容
    • 标准浏览器 => onload 事件
    • IE 低版本 => 没有 onload 事件
      -> IE 低版本只有 onreadstatechange 方法
      -> IE 低版本接受响应只能使用 onreadystatechange 这个事件
    • 在这个事件里面进行判断
      • 判断 ajax 结束了 -> ajax 状态码为 4 就是结束
      • 判断 本次 http 请求时成功了的 -> http 状态码时 200 ~ 299 之间
        -> xhr.status 就是本次请求的 http 状态码
