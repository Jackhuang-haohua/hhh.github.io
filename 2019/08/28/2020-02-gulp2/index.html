使用gulp

发表于 2019-08-28 更新于 2020-05-17 分类于 gulp
本文字数： 3.2k 阅读时长 ≈ 6 分钟
打包html，css，js，其他文件，配置服务器，代理，任务流

gulp

假设项目有这些页面：

  • pages文件夹 -> 存放html页面
  • css文件夹 -> 存放css文件
  • js文件夹 -> 存放js文件
  • mages文件夹 -> 存放图片文件
  • fonts文件夹 -> 存放字体图标文件
  • videos文件夹 -> 存放视频文件

我们在根目录下新建gulpfile.js文件，vscode会出现一个小辣椒图标；

gulp一些方法

  1. src()：src('你要打包的文件的地址')
  2. pipe()：管道，是任务可以继续“流”向下一步
  3. dest()：用来写入文件，把已经压缩号的代码放在那一个文件夹里面；如果没有你指定的文件夹，会自动创建一个这个文件夹放进去
  4. parallel()：用来并行执行多个任务，gulp.parallel(任务1，任务2, ...)
  5. series()：用来逐个执行多个任务，gulp.series(任务1，任务2，...)
  6. watch()：用来监控文件变化的，gulp.watch(监控的文件目录，要执行的任务)

del

　最好就是在每次整体打包之前, 把 dist 目录删除了，用最新的 src 重新生成一遍内容；

webserver

　在开发过程中直接把写的东西在服务器上打开；自动刷新: 当 dist 目录里面的代码改变以后, 就会自动刷新浏览器；

自动监控

监控 src文件夹 下的文件, 只要一修改, 就执行对应的任务

导出任务流

　导出一个默认任务，后面在命令行执行 gulp default 的时候, 就会自动把写在 parallel 里面的五个任务给一起执行了；
　tip：当在命令行执行 gulp default 的时候, 可以不写 default
