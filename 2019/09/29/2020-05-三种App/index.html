了解目前流行的3种App

发表于 2019-09-29 更新于 2020-05-06 分类于 App
本文字数： 1.1k 阅读时长 ≈ 2 分钟
原生App，webApp，混合App；写的比较乱，了解到什么就写什么

一. NativeApp(原生App)

纯工具类的App，比如微信客户端、手机QQ；
特点：

  1. 开发：由原生Native人员开发的、安卓（Android）、苹果（IOS）
  2. 语言：Android（Java + XML）、IOS (Objective-c / Swift + XML)
  3. 更新维护：版本更新、打补丁包、需要从 app store 进行应用更新
  4. 使用：必须去app-store里面进行下载更新才可以使用
  5. 优点：运行稳定、用户粘度高、安装使用比较方便、用户体验强、可以操作原生功能
  6. 缺点：不能跨平台、必须开发两套代码（Android+IOS）

二. WebApp(移动M站)

​特点：

  1. 凡是可以在移动端浏览器里面进行打开的网站都称之为WebApp
  2. 开发：H5前端开发工程师
  3. 语言：html5+css3+js(zepto+react+angular+vue等)
  4. 更新维护：直接无痕更新（浏览器缓存的文件是根据路径进行缓存的，所以前端会在地址栏后面加上一串随机码，以保证每次打开页面都是重新请求，不会用旧的缓存来渲染）
  5. 使用：无需下载、直接通过用户的手机浏览器（safari)可以直接打开进行访问
  6. 优点：可以跨平台的、开发成本低、使用成本低、更新维护成本低
  7. 缺点：不稳定、流畅度差、用户体验差、不能调用原生设备

三. Hybrid App (混合App)

  1. 在native的壳子中，通过webview，在webveiw中内嵌H5的页面即可。内部通过JSBridge这个技术实现了原生与JS的交互。

  2. 有两种开发模式

    • 原生主导的开发模式：
    • 整个app大部分的界面都是由原生人员开发的，但是某一些经常变动的页面由内嵌的H5的页面去实现的。
      原生人员打包app (AndroidStudio or Xcode)

  3. H5主导的开发模式：

    • h5可以借助一些已经赋予功能的一些打包工具，实现前端软件的打包发布。
    • 检测一下哪些是原生native应用，哪些是hybrid应用，主要看内部有没有内嵌的h5的页面即可。

  4. 同类型的产品只需要看安装包的大小，如果小的话肯定就是hybrid应用。android手机上面有调试模式，内部就会显示布局界面。如果应用是native开发的，就会被线条所包裹，从而划分。

  5. 因为有一些页面，可能没过几天就需要进行更新、变动。如activity活动页面，倘若让原生进行开发的话，需要花费太长的周期，所以采用H5页面内嵌到webview中，这样就可以大大的提高我们的开发效率。并且很多公司都在采用这样的开发模式。

  6. 例如美团，我们可以在美团的app中，对于猫眼电影的这部分业务逻辑，内嵌早已经开发好的webapp了。这样的话，美团就不需要对猫眼这部分的业务逻辑进行重新构建了。
