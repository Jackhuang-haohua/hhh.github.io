闭包学习总结

发表于 2020-02-23 分类于 JavaScript
本文字数： 969 阅读时长 ≈ 2 分钟
利用Js基础知识做出轮播图

一、前言

闭包，似乎是非常抽象的概念，但又经常出现本人在上周学习完之后，有了一定的理解，在这里分享给大家

二、闭包的概念

为了更好的理解，我们直接看一段demo

用自然语言描述如下：

  • 定义普通函数 A
  • 在函数 A 中定义变量 num
  • 在 A 中定义普通函数 B
  • 在函数 B 中使用变量 num
  • 在 A 中返回 B
  • 执行 A，并把 A 的返回结果赋值给变量 C
  • 执行 C

一句话总结：当一个内部函数被其外部函数之外的变量引用时，就形成了一个闭包。

闭包的生成有三个必要条件(缺一不可)

  • 在函数 A 内部直接或者间接返回一个函数 B
  • B 函数内部使用着 A 函数的私有变量(私有数据)
  • A 函数外部有一个变量接受着函数 B

此时，就形成了一个不会销毁的函数空间–闭包空间，我们管这个不会销毁的 a 函数的执行空间叫做 闭包空间，把函数 a 里面返回的 函数 b, 叫做函数a 的 闭包函数。

三、闭包的特点

闭包形成后，有几个特点，如下

1. 延长了变量的生命周期

  • 优点: 因为执行空间不销毁, 变量也没有销毁
  • 缺点: 因为执行空间不销毁, 会一直存在在内存中

2. 可以访问函数内部的私有变量

  • 优点: 利用闭包函数可以访问函数内部的私有变量
  • 缺点: 执行空间不会销毁, 会一直存在在内存中

3. 保护私有变量(只要是函数, 就有这个特点)

  • 优点: 保护私有变量不被外界访问
  • 缺点: 如果向访问, 必须要利用闭包函数

虽然闭包优点多多，但我们不能无所顾忌的使用，因为闭包存在一个致命的缺点

当一段内存空间中有一个不会被销毁的东西一直存在，那么就会出现内存占用的现象, 如果过多, 就会导致内存溢出，其结果就是内存泄漏

四、闭包的作用

在JS里面，一个函数能访问的有：

  • 本身的参数
  • 局部变量或函数
  • 外部变量：全局变量，DOM，外部函数等

因此，当你需要延长变量的声明周期，或者你需要访问某一个函数内部的私有数据的时候，当我们需要在模块中定义一些变量，并希望这些变量一直保存在内存中但又不会 “污染” 全局的变量时，就可以使用闭包。
但是，如果有别的解决方法, 尽量不要使用闭包函数！

这只是对闭包的简单了解，闭包还有联合匿名函数等高级写法，但本人尚未深入了解，在下次学习之后，定会再次分享。
