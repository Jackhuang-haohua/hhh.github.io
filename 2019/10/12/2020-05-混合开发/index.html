混合开发

发表于 2019-10-12 更新于 2020-05-08 分类于 App
本文字数： 6.3k 阅读时长 ≈ 11 分钟
在原生Android上，嵌入html页面；环境都是同事帮忙配置的，sdk搞了半天都没搞定TOT

一、原生语言

　苹果手机：ios系统，由swift和c++/object-c语言编写，后缀名为ipa（ios开发的安装包的后缀名），swift语言是一种开源的语言；
　安卓手机：android系统，由java（android）语言编写，后缀名为apk
　wp手机：windowphone系统，由c#语言编写， 一般说到混合开发不考虑这个版本

二、混合开发

　即原生语言开发+js的开发；
　js无法调动系统的原生功能（拍照、短信、打电话、通讯录），但是原生（android、ios、wp）可以
原生有很大的适配问题（特别是android），且更新麻烦（比如频繁更新活动，原生不好做），而js可以很好的解决这个问题；

三种App
　hybridapp：js+android/ios，可以安装至手机应用；
　webapp/m站：js完成，touch端项目，浏览器上访问的手机网站；
　nativeapp：android/ios，可以安装至手机应用；

四、新建项目

　会遇到一些选项；

  • Application name：应用名称
  • Company domain：公司标识
  • Package name：应用包名，唯一标识；提示应用更新，数据统计等会用到
  • Minimum SDK：指定手机系统，基于API X开发
  • Phone and Tablet / wear / TV ：开发的应用在什么设备运行

　前端只需要按照项目需求编写页面，假设你的一个按钮需要调用系统的功能（部分功能需要添加权限、摄像头、存储照片至sd卡权限）;

　同时前端也会定义一些方法，但是前端自己不去调用，android会根据webview的loadUrl方法进行调用，并且传参，进一步调用前端的代码（具体如何做根据需求去看）；
　比如前端给后端传一个方法A -> 后端调用A并传参ID -> 前端收到ID -> 根据这个ID来请求详情页;
